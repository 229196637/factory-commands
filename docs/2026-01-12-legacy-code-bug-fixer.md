## Skill: legacy-code-bug-fixer (遗留代码Bug修复)

### 位置选择
**Personal skill** (`C:\Users\unclefata\.factory\skills\legacy-code-bug-fixer\`)

理由：这是一个通用的bug修复工作流，适用于任何项目中没有AI辅助时编写的遗留代码，不限于特定项目。

### 技能目的
处理之前没有AI辅助时编写的代码中出现的逻辑bug，提供系统化的分析和修复流程。

### 核心工作流程

**Phase 1: 问题收集**
- 收集错误信息（日志、堆栈、错误描述）
- 确定问题发生的上下文和触发条件

**Phase 2: 深度分析**
1. 定位错误代码位置
2. 分析调用链条（从入口到错误点）
3. 理解代码原始意图和设计
4. 识别逻辑缺陷（不是简单nil检查）

**Phase 3: 方案设计**
- 提供多个修复方案（A/B/C）
- 每个方案列出优缺点
- 评估对现有逻辑的影响
- **等待用户确认后再执行**

**Phase 4: 修复验证**
- 实施选定方案
- 验证修复效果
- 检查是否引入新问题

### 特殊处理
- **缺失方法**：对比原项目（如Unity原项目）查找实现
- **异步问题**：检查回调时机和变量初始化顺序
- **缓存问题**：检查缓存命中时的同步回调

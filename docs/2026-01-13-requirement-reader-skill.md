## Skill: requirement-reader

### 功能描述
读取项目根目录下 `docs/requirement/<模块名>/` 文件夹中的需求文档，帮助理解模块的需求和规则。

### 触发场景
- 用户说"阅读需求"、"查看需求"、"读取需求文档"
- 用户说"了解XX模块的需求"
- 开发新功能前需要理解需求规范

### 技能目录结构
```
.factory/skills/requirement-reader/
└── SKILL.md          # 主技能文件
```

### SKILL.md 内容概要

**Phase 1: 定位需求文档**
1. 确认 `docs/requirement/` 目录存在
2. 根据用户指定的模块名定位对应文件夹
3. 列出文件夹中所有文档文件

**Phase 2: 阅读需求**
1. 按优先级阅读文档（README.md > 其他.md > .txt）
2. 提取关键信息：功能需求、业务规则、技术约束、接口定义

**Phase 3: 输出报告**
1. 生成需求摘要
2. 列出关键规则和约束
3. 标注需要注意的边界情况

### 边界情况处理
| 情况 | 处理方式 |
|------|----------|
| 模块文件夹不存在 | 列出可用模块，提示用户选择 |
| 文件夹为空 | 提示无需求文档 |
| 多个文档文件 | 按优先级依次阅读并整合 |
| 包含子文件夹 | 递归读取所有文档 |

### 位置
**Project skill**: `.factory/skills/requirement-reader/SKILL.md`

理由：`docs/requirement/` 是项目特定的目录结构，应随项目共享。

### 验证方式
创建后可通过 `/skills` 查看是否列出，然后用"阅读XX模块需求"测试调用。

## 参数说明

格式: `/api-analyze [模块路径或名称]`
- 指定模块路径或名称进行分析
- 省略则分析当前上下文相关的模块

## Instructions

1. 确定要分析的模块:
   - 如果提供了路径/名称，分析指定模块
   - 如果未提供，从上下文推断相关模块

2. 调用 `api-analyzer` droid 进行分析:
   - 使用 Task 工具启动 `api-analyzer`
   - 传入模块路径和分析要求

3. 输出接口清单:
   - 公开接口/函数列表
   - 参数和返回值
   - 使用示例
   - 依赖关系

4. 结果不保存文件，直接输出显示
